# ADR-0002：v0.2 后端纵切（NestJS + Prisma 服务端，shared 成为三端同构事实源）

- 状态：**Accepted（M1 已落地）** — 人类以 v0.2 任务书立项；后端纵切分 M1→M4 串行交付。
- 日期：2026-06-26
- 影响里程碑：M1（服务端骨架 + shared 接缝整理，本次落地）→ M2（LLM Gateway）→ M3（审批/Timeline 持久化）→ M4（工具执行：幂等 + 回滚）
- 关联：`docs/PRD.md` §3 / §5.2 / §9.3 / §11 / §13；v0.2 任务书 §0/§2/§5；`docs/DECISIONS.md`；`docs/adr/0001-adopt-react-19.md`

> 本 ADR 记录 v0.2 的 5 项关键决策。标注 **[M1 已落地]** 的本次实现；标注 **[Mn 待落地]** 的是已定方向、留对应里程碑实现，先在此固化以防漂移。

---

## 0. 是否违反 frozen spec？（先过护栏）

**不违反——这是一次人类显式立项的 v0.2 范围扩展，不是 Agent 擅自扩范围。** `docs/PROGRESS.md` 早已把「真实 LLM / 真实持久化 / 跨端同步」列为 frozen v0.1 之外、**需人类立项**的增强；本次任务书即该立项。v0.1（sealed `v1.0.0`）的所有红线在 v0.2 仍全程有效：不碰登录/RBAC/多租户/真实 CRM；shared 继续零 React/UI；承重墙（类型/Zod/XState/分流）复用而非重建；服务端是 shared 的**第三个消费者**，不另起平行规则。

---

## 1. 数据权威：服务端为持久状态与 Timeline 权威源，客户端为乐观层 **[M1 建表与连接；M3 全量落地]**

- **背景**：v0.1 审批状态与 Timeline 只在客户端内存，刷新即失、跨端不一致。v0.2 要把权威迁到服务端。
- **决策**：`apps/server`（NestJS + Prisma + SQLite）持久化 `ToolAction` 状态与 `TimelineEvent`，是唯一权威源。客户端 XState 实例降为**交互/乐观层**：本地转移后向服务端 POST「意图转移」，服务端用 **shared 同一份** `approvalMachine` 二次校验转移合法性后再持久化，以服务端返回为准。
- **备选**：(a) 客户端继续权威 + 仅上报日志——无法满足「重启不丢、跨端一致」；(b) 完全放弃客户端状态机、纯服务端渲染——丢掉 v0.1 承重墙叙事且改动过大。均排除。
- **风险**：**双事实源**是本次最大架构风险（客户端乐观态与服务端权威态分叉）。**缓解**：转移合法性只认服务端（同一份 XState 定义二次校验），客户端失败即回滚乐观 UI；M1 阶段先把 Prisma 列与 shared schema 的**字段对齐用测试钉死**（见 §5），避免地基漂移。
- **回滚预案**：M3 之前客户端仍可独立运行（mock store 未拆除）；服务端不可用时前端退回 v0.1 内存模式即可。

## 2. LLM provider 选型 / key 处理 / SSE / 确定性隔离 **[M2 待落地]**

- **背景**：M2 要把 Mock 升级为真模型出建议，但服务端校验与测试必须保持确定性。
- **决策（已定方向）**：服务端实现 `RealLLMAdapter`（实现 shared 的 `LLMAdapter` 接口），用 `LLM_PROVIDER=mock|real` 环境变量切换；真模型输出仍过 **shared 的 `AIAnalysisSchema`**，非法即走现有 fallback。key 走 `.env`（**绝不入库、不提交**）。streaming 用 **NestJS 原生 SSE**，不引 Vercel AI SDK。
- **备选**：Vercel AI SDK / LangChain——超出「治理层」诉求且引入额外抽象，排除（与 v0.1 §18.2 一致）。
- **风险**：真模型非确定性污染测试。**缓解**：契约测试一律跑在 `mock` provider 上；真模型只做本地 smoke、**不入 CI**（CI 无外部 LLM、无需真 key，见 §4 与 `ci.yml`）。
- **回滚预案**：`LLM_PROVIDER=mock` 一键退回确定性链路。

## 3. 幂等策略 **[M4 待落地]**

- **背景**：批准后服务端真调 Mock CRM，重复请求不得产生重复副作用。
- **决策（已定方向）**：`POST /actions/:id/execute` 带 **idempotency key**；服务端按 key 持久化执行结果，重复 key 直接返回首次结果、不重复触达 Mock CRM。粒度＝单个 ToolAction 的一次执行。
- **风险/缓解**、**回滚预案**：M4 落地时在本节补全（背景/决策/备选/风险/回滚）。

## 4. 回滚语义：补偿动作 vs 状态回退 **[M4 待落地]**

- **背景**：执行失败或人工触发需进入 `rollback` 终态并留痕。
- **决策（已定方向）**：`rollback` 通过**补偿动作**实现（各 ToolAction 类型定义各自补偿），而非简单状态回退；补偿过程写入 Timeline。复用 shared `approvalMachine` 既有 `failed--ROLLBACK-->rollback` / `success--ROLLBACK-->rollback` 转移，不新增状态。
- **风险/缓解**、**回滚预案**：M4 落地时补全；各动作类型的补偿定义在该里程碑给出。

## 5. shared 同构化（禁 Node 内置）+ 服务端如何消费 shared 的 TS 源码 **[M1 已落地]**

- **背景**：服务端（Node）成为 shared 第三消费者。shared 必须在 **Vite / Metro / Node** 三端同构可消费，故除既有「零 React/UI」外，**追加禁止 import Node 内置**（`fs`/`path`/`process`/`crypto`/`node:*` 等）。
- **决策**：
  1. `LLMAdapter` 接口从 `mock-ai` **迁入 `packages/shared/src/llm-adapter.ts`**（只引用 shared 自身数据类型，零外部依赖）；`mock-ai` 的 `MockLLMAdapter` 退化为 `= LLMAdapter` 别名实现，既有消费者零破坏。
  2. ESLint `no-restricted-imports` 对 `packages/shared/**` **加严**含全部 Node 内置 + `node:*`；新增 `packages/shared/src/eslint-boundary.test.ts` 作**反例测试**（shared 路径下 `import 'node:fs'` 必须被规则拦截），把红线行为化。
  3. 服务端运行时消费 shared 的**原始 TS 源**（沿用全仓「无构建、源码消费」模型，不给 shared 加 build）：用 `ts-node`（`module: commonjs` + `experimentalDecorators` + `emitDecoratorMetadata`，保留 NestJS DI 元数据）+ `tsconfig-paths/register`。
- **备选**：
  - 给 shared 增构建产物（dist + 条件 exports）供 Node 消费——改动 shared 既定消费模型、波及 web/mobile，排除。
  - 用 `tsx`/esbuild 跑 server——**不产出装饰器元数据**，破坏 NestJS DI（M2+ 致命），排除。
  - SWC（`@swc-node/register` / `nest start -b swc`）——支持元数据，**保留为回退方案**。
- **风险/已遇并解决**：ts-node 的 CJS 加载器遵循 shared 的 `package.json "type":"module"`，直接 `require()` 其 `.ts` 抛 `ERR_REQUIRE_ESM`。**解法**：ts-node `moduleTypes` 覆盖，把 `../../packages/{shared,mock-ai}/src/**/*.ts` 强制按 `cjs` 加载（见 `apps/server/tsconfig.json`）。已 smoke 验证：Nest 正常启动、运行时成功消费 shared（启动日志打印 shared 的审批阈值）、`GET /health` 返回 `{"status":"ok"}`。
- **回滚预案**：若后续某环境 ts-node 仍摩擦，切 SWC 回退方案，仅改 server 的 dev/start 脚本与少量 devDep，不动 shared 与其余四包。

## 7. 诚实边界（简历口径）

完成后可写「真实 LLM + 服务端结构化输出校验 + 持久化审计日志 + 带回滚的工具执行闭环」；**不可**写「生产环境 / 真实用户 / 接入真实 CRM」。Mock CRM 仍是 mock，如实表述。
